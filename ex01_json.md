# Упражнение 1: Получение структурированного JSON (Get Structured JSON)

**Цель:** Заставить модель отключить текстовую генерацию и вернуть данные в строго заданном машиночитаемом формате (JSON).

**Мой промпт (с указанием точной схемы):**
> Проанализируй следующую функцию на Go. 
> Верни ТОЛЬКО валидный JSON, без словесного мусора, комментариев или форматирования markdown (без ```json). 
> 
> Ты должен строго соблюдать следующую структуру:
> {
>   "function_name": "string",
>   "parameters": [{"name": "string", "type": "string"}],
>   "return_type": "string",
>   "has_loops": boolean
> }
> 
> Код:
> func AppendRange(min, max int) []int {
>     if min >= max { return nil }
>     var result []int
>     for i := min; i < max; i++ {
>         result = append(result, i)
>     }
>     return result
> }

**Ответ модели (Output):**
{
  "function_name": "AppendRange",
  "parameters": [
    {
      "name": "min",
      "type": "int"
    },
    {
      "name": "max",
      "type": "int"
    }
  ],
  "return_type": "[]int",
  "has_loops": true
}

**Вывод (Почему программе нужен JSON, а не обычное предложение?):**
Программе (бэкенду) нужен JSON, потому что алгоритмы парсинга требуют строгой, предсказуемой структуры пар «ключ-значение» для маппинга данных в переменные, тогда как естественный язык непредсказуем и вызовет критическую ошибку при обработке кодом.

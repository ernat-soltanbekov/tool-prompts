# Упражнение 3: Сборка полной цепочки (Chain It Into an Answer)

**Цель:** Соединить запрос пользователя, вызов инструмента (JSON) и финальный ответ на основе полученных данных в единую цепь.

**Шаг 1: Запрос пользователя**
> Братан, что там по градусам сейчас в Астане?

**Шаг 2: Вызов инструмента моделью (из Упражнения 2)**
> {"tool":"get_weather","args":{"location":"Астана"}}

**Шаг 3: Передача фейкового результата от бэкенда и финальный промпт**
> Пользователь спросил: "Братан, что там по градусам сейчас в Астане?"
> Ты ранее запросил вызов инструмента `get_weather` для локации `Астана`.
> 
> Вот сырые данные, которые вернул мой код (бэкенд) от API погоды:
> {"status": "success", "data": {"temp": "+28", "condition": "Ясно, без осадков"}}
> 
> Твоя задача: используя эти данные, сформулируй финальный ответ для пользователя. Отвечай в том же стиле, в котором был задан вопрос. Никакого JSON, просто нормальный текст.

**Финальный ответ модели:**
> Братан, сейчас в Астане +28 °C. 😎 На улице ясно, без осадков, так что погода вообще кайф — можно спокойно гулять или выбираться по делам без зонта.

**Вывод (Где эта цепь может сломаться, и что нужно проверять?):**
Эта цепь может сломаться на этапе парсинга, если ИИ выдаст невалидный JSON, выдумает несуществующий аргумент, или если стороннее API погоды упадет и вернет ошибку (бэкенд должен проверять формат JSON, валидировать аргументы и обрабатывать таймауты API перед возвратом данных модели).

# Упражнение 2: Выбор инструмента моделью (Let the Model Pick the Tool)

**Цель:** Научить модель работать в качестве маршрутизатора (роутера), который анализирует запрос пользователя и выбирает нужную функцию (инструмент) с правильными параметрами.

**Промпт (Инструкция для модели):**
> У тебя есть доступ к двум инструментам:
> 1. `get_weather(location)` -> возвращает текущую погоду для указанного места.
> 2. `define_word(word)` -> возвращает словарное определение слова.
> 
> Твоя задача — выбрать правильный инструмент на основе запроса пользователя. 
> Ответь ТОЛЬКО объектом JSON, указав название инструмента и его аргументы. Строго используй такую структуру: {"tool": "название_инструмента", "args": {"аргумент": "значение"}}. Больше ничего не пиши.

**Запрос 1:**
> Братан, что там по градусам сейчас в Астане?

**Ответ модели 1 (JSON):**
{"tool":"get_weather","args":{"location":"Астана"}}

**Запрос 2:**
> Что означает слово рекурсия?

**Ответ модели 2 (JSON):**
{"tool":"define_word","args":{"word":"рекурсия"}}

**Вывод (Правильно ли ИИ выбрал инструмент и в чем заключается работа моего кода?):**
Модель безошибочно распознала намерение пользователя и выбрала нужный инструмент в обоих случаях. Теперь задача моего кода (бэкенда) — принять этот JSON, распарсить его, увидеть команду `get_weather` или `define_word`, выполнить реальный запрос к погодному API или словарю, и вернуть полученные данные обратно нейросети.

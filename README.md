# Проект: Tool Prompts (Работа с инструментами)

## Описание
Этот проект демонстрирует архитектуру Function Calling (вызов функций). ИИ-модели не умеют самостоятельно выполнять код, обращаться к базам данных или делать запросы в интернет. Приложение работает за счет того, что модель работает как "маршрутизатор": она принимает естественный язык, решает, какой инструмент нужен, и отдает бэкенду команду в виде JSON.

## Главные выводы (Main Takeaways)

* **Почему для работы с инструментами важен структурированный вывод (JSON)?** Программе на бэкенде нужен жестко структурированный формат (JSON), потому что алгоритмы парсинга требуют точной структуры пар "ключ-значение" для маппинга данных в переменные; парсинг естественного непредсказуемого текста приведет к критической ошибке (crash).
* **Какова зона ответственности кода?** За фактическое выполнение задачи (отправка HTTP-запроса, вычисление, работа с БД) и обработку системных ошибок отвечает исключительно мой код (бэкенд), в то время как ИИ отвечает только за генерацию текстовой команды на использование этого инструмента.
